The Birthday Paradox Simulation

The Birthday Paradox explores the mathematical logic that in a room of just 23 people, there is a 50% chance that two people share a birthday.

This project is a java application that simulates experiments for group sizes ranging from 5 to 100 people. Using a random birthday generator, the program runs 20 trials for each group size to calculate the percentage of shared birthdays and outputs the results in a formatted table.

The project follows and object-oriented design consisting of four primary classes: Simulation, Experiment, Trial, and Birthday. This modular structure allows the Simulation class to manage the high-level logic and random number generation, while the Experiment class handles specific group sizes and trial counts. By separating these concerns, the code remains organized and follows clean code principles.

I was responsible for implementing the core simulation logic, including the Trial.java and Birthday.java classes. My partner and I followed clean code practices best we could and collaborated on the project via git version control.

Throughout this project, I improved my skills in test driven development and learned how to use Intellij debugging tools to verify results and manage dependencies.
