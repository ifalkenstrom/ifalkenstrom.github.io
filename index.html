Birthday Paradox Simulation

This Java-based simulation explores the mathematical Birthday Paradox, which calculates the probability of shared birthdays within varying group sizes.

The application runs experimental trials for groups of 5 to 100 people, utilizing a random generator to simulate birthdays and calculate shared-date percentages over 20 trials per group.

Following clean code and TDD principles, I implemented modular classes for Simulation, Experiment, and Trial to maintain a clear separation of concerns and ensure mathematical accuracy.

Project Logic & Requirements

The system functions as a discrete-time engine processing passenger requests across Standard, Freight, Penthouse, and Express elevators. Elevators manage floor movement (1 floor per round), idle states during pickups, and direction changes, with an automated return to Floor 1 when inactive. The program is initialized via 7 command-line arguments to specify building parameters.

OOP Principles & Design

I utilized an abstract Elevator class to define shared state while allowing subclasses to override canAcceptCall() for unique policies. This design adheres to SOLID principles—specifically the Open-Closed Principle—by enabling the addition of new elevator types without modifying core simulation logic.

My Implementation Role

I personally developed the Simulator engine and Scheduler interface. My runSimulation logic coordinated unit movements and real-time state output. I followed Test-Driven Development (TDD), authoring JUnit tests to verify complex edge cases like simultaneous picks.
